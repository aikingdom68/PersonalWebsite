Glossary
先把 4 个"黑话"翻译成人话
Agent
专职数字员工
只做一件事的虚拟专职员工
MD 文档
岗位 SOP / 员工手册
它必须遵守的"宪法"
Skills
能力包 / MCP 集合
把各种外部能力（读表、搜索、调用 API 等）打包成可插拔的工具集合，类似 MCP 的能力封装
Workflow
流水线传送带
强制把上一步输出送进下一步

Chain Rule
AI 翻车背后的底层规律：链式法则
90%
每一步正确率 90%
看起来已经不错了
59%
连续 5 步后
总正确率只剩 59%
接近抛硬币
35%
连续 10 步
可能跌到 35%
越长的链路，越容易滚雪球式失真
11 / 26
Solution
Agent + Workflow
的价值在于
"斩断误差链"
每个节点只做一件事
并接受边界校验
每个 Agent 聚焦一项极细分任务
单点正确率更容易调优到 95%+
节点之间可配置校验与人工确认
不是让错误一路滚大，而是逐段稳定输出
12 / 26
Symbiosis
Workflow 是路，Agent 是车
没有 Workflow 的 Agent
像无头苍蝇一样乱飞
没有固定路线
输出不可预测
没有 Agent 的 Workflow
出错后难以定位是哪个节点的问题
找到了也不知道怎么改，规则变更成本高
想在某节点暂停审核再继续，实现很麻烦
结论
路和车必须同时设计，割裂理解一定会误用 AI。
